Inheritance

Objects inherit methods and properties from other Objects prototypes. This is configured at startup, so if you change this, you must restart the instance for changes to take effect. Other methods (which are overwritten by the child) are available on the _super object. For instance: 

    function User(_id) { 
        this._id = _id; 
        return this;
    };
    User.prototype = { 
        constructor : User,
        properties : { 
            inherits : "Person"
        }
    } 
    
    function Person(_id) { 
        this._id = _id;
        return this; 
    }; 
    Person.prototype = {
        constructor : Person,
        properties : { 
            hair_colour: 'red', 
        } 
    }

In the above example, because User inherits from Person it will receive red hair.

Conventions (code)

All classes are lower case seperated by underscores, this is because various aspects of the application automatically convert names to lower case unexpectedly.
_ a class that should not be used. It is dangerous to use and is in general either extended from, or used singly. can be thought of as an abstract object
$ a global class that can be used by all objects (including ajax endpoints)

Conventions (file)

$ at start of filename, indicates a class used solely by the main 'boss.js' script.
lower_case all filenames are lower case seperated by underscores, this is because various aspects of the application automatically convert names to lower case unexpectedly.

Global objects ($)

$server a class which contains all aspects of the server.
Public methods
  • startup: starts it up
  • restart: restarts safely, re-instantiating all objects. Relatively expensive
  • shutdown: shuts it down
  • childProcessCommand: can run a command against the OS
  • readConfig: reads the config file from the './config.json' location
Properties
  • asset_manager: controls all files. contains a record of all files installed, and can minify and productionise files upon reqest.
  • plugin_manager: controls installed plugins, and installs/uninstalls plugins upon request.
  • client_manager: controls all files in the client - client-side JS, CSS etc and makes sure they are included in the index page.
  • angular_manager: controls the root Angular 'App' file, and all the angular routing, constants, and setup.
  • route_manager: Should never be directly interacted with. By itself it watches for assets to be created and then creates routes automatically for them, depending on their type.
  • http_server: The web server instance for this application, can be restarted using methods within the class.
$cache a class which can contain any cached object.
Public methods
  • getAll: gets all cache items currently stored.
  • get: get a single cache item, or a cache group. Cache items are dot-notated, like an object syntax.
  • unset: unsets a single item or group, using dot-notation.
  • set: sets an object. Bear in mind that objects will simply overwrite previously held objects.
  • flush: empties the entire cache.
  • setup: called upon installation, not really useful.
Properties
  • _cacheObject: the cache store. Don't touch.
$dir a class which enables the instantiation of any object from any other object location.
Public methods
  • no methods. To instantiate a new object from the flow_server directory, use "new $dir.[Object](params)"
Properties
  • List of properties is extensive, being every class available to the application.
$dbi a class containing all of the Database schemas, allowing for universal access to the database.
Public methods
  • no methods. Object should be directly called to retrieve a database table for further use.
Properties
  • no properties.
